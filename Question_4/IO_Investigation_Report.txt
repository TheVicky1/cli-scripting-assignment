FILE ACCESS AND I/O INVESTIGATION REPORT

1. Application Log Creation

Command:
touch application.log
ls -l application.log

Observation:
The touch command created an empty file named application.log.
The ls -l command displayed its permissions, ownership, size, and modification time.

2. Writing to a Log Using File Descriptor 3

Commands:
exec 3>>application.log
echo "Application started successfully" >&3
echo "Log entry written through file descriptor 3" >&3
cat application.log

Output:
Application started successfully
Log entry written through file descriptor 3

Observation:
The exec command opened application.log in append mode using file descriptor 3.
The two echo commands wrote data into the log through descriptor 3 without overwriting existing contents.

5. Closing File Descriptor 3

Command:
exec 3>&-

Observation:
The command closed file descriptor 3.
After it was closed, application.log no longer appeared as descriptor 3 in /proc/$$/fd or lsof output.

6. Standard Output Redirection

Commands:
echo "This is standard output from the application" > stdout.txt
cat stdout.txt

Output:
This is standard output from the application

Observation:
The greater-than operator redirected standard output, file descriptor 1, into stdout.txt.
The output was saved in the file instead of being displayed directly in the terminal.

7. Standard Error Redirection

Commands:
ls /directory_that_does_not_exist 2> stderr.txt
cat stderr.txt

Output:
ls: cannot access '/directory_that_does_not_exist': No such file or directory

Observation:
The invalid ls command generated an error through standard error, file descriptor 2.
The 2> operator redirected the error message into stderr.txt.

8. Separate Output and Error Redirection

Command:
ls /etc /directory_that_does_not_exist > normal_output.txt 2> error_output.txt

Normal output:
The valid listing of the /etc directory was stored in normal_output.txt.

Error output:
ls: cannot access '/directory_that_does_not_exist': No such file or directory

Observation:
Standard output was redirected into normal_output.txt.
Standard error was redirected separately into error_output.txt, showing that Linux manages them as independent streams.

9. Combined Output and Error Redirection

Command:
ls /etc /directory_that_does_not_exist > combined_output.txt 2>&1

Observation:
The normal directory listing and the error message were both stored in combined_output.txt.
The expression 2>&1 redirected standard error to the same destination as standard output.

10. Overwrite and Append Redirection

Commands:
echo "First log entry" > redirection_log.txt
echo "Second log entry" >> redirection_log.txt
cat redirection_log.txt

Output:
First log entry
Second log entry

Observation:
The > operator created or overwrote the destination file.
The >> operator appended the second entry without deleting the existing contents.

12. Explanation of Linux File I/O

Linux represents open files and other input/output resources using numeric file descriptors.

The standard descriptors are:
0 — Standard input
1 — Standard output
2 — Standard error

Additional files opened by a process normally receive descriptor numbers 3 and above.

The Linux kernel maintains information about each open file, including its access mode, current offset, and reference count. Shell redirection changes the destination associated with a file descriptor.

The > operator redirects standard output and overwrites the destination file.
The >> operator appends output to a file.
The 2> operator redirects standard error.
The 2>&1 expression redirects standard error to the same destination as standard output.

Utilities such as lsof and the /proc file system help identify open files and file descriptors. The ulimit command displays resource restrictions applied to processes.
